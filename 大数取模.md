### 你做错了什么

1. **模数抄错了**
   题目要求：

   [
   10^9+7=1000000007
   ]

   你写成了：

   ```cpp
   #define MOD 1000000009
   ```

   因而 `n=35` 时，你的输出正好是答案对 (10^9+9) 取模的结果。

2. **取模后使用普通整数除法**
   你原来写的是：

   ```cpp
   (p1 % MOD) / (p2 % MOD)
   ```

   一般而言：

   [
   \left(\frac ab\right)\bmod M
   \neq
   \frac{a\bmod M}{b\bmod M}
   ]

   取模会丢失整数商所需要的信息。模意义下的除法应当转化为乘模逆元：

   [
   \frac ab\bmod M
   ===============

   a\cdot b^{-1}\bmod M
   ]

3. **连乘过程中没有及时取模，导致溢出**
   你先连续计算：

   ```cpp
   p1 *= n - i;
   ```

   最后才 `% MOD`。当 (n) 较大时，中间结果会迅速超过 `unsigned long long` 的范围。

   溢出后再取模，不能恢复原数的正确余数。应改为：

   ```cpp
   p1 = p1 * (n - i) % MOD;
   ```

4. **你的组合计数式本身没有错**
   对于固定的 (k)，方案数为：

   [
   \binom{n-1}{k}(k-1)(n-k-1)
   ]

   经过手动约分后：

   [
   \frac{(n-1)(n-2)\cdots(n-k-1)}
   {k(k-2)!}
   ]

   这与你代码中构造的分子、分母一致。问题主要发生在计算机实现，而不是组合数学推导。

### 相关知识

#### 加法、减法、乘法可以中途取模

加法：

[
(a+b)\bmod M
============

((a\bmod M)+(b\bmod M))\bmod M
]

乘法：

[
ab\bmod M
=========

((a\bmod M)(b\bmod M))\bmod M
]

减法：

[
(a-b)\bmod M
============

((a\bmod M)-(b\bmod M))\bmod M
]

在 C++ 中为防止负数，通常写成：

```cpp
(a - b + MOD) % MOD
```

前提是 `a` 和 `b` 已经位于 `[0, MOD)`。

因此，对于只包含加法和乘法的式子，**多次取模与最后一次取模在数学上等价**；程序中多次取模可以避免中间结果溢出。

#### 普通除法不能直接取模

一般不成立：

[
(a/b)\bmod M
============

((a\bmod M)/(b\bmod M))\bmod M
]

在模意义下，应使用：

[
a/b\equiv a\cdot b^{-1}\pmod M
]

其中模逆元 (b^{-1}) 满足：

[
b\cdot b^{-1}\equiv1\pmod M
]

逆元存在的条件是：

[
\gcd(b,M)=1
]

当 (M) 是质数，且 (b\not\equiv0\pmod M) 时，根据费马小定理：

[
b^{-1}\equiv b^{M-2}\pmod M
]

可用快速幂求出：

```cpp
inverse = qpow(b, MOD - 2);
```

#### 取模不能挽救已经发生的溢出

数学上有：

[
(ab)\bmod M
===========

((a\bmod M)(b\bmod M))\bmod M
]

但 C++ 执行：

```cpp
a * b % MOD
```

时，会先计算 `a * b`。如果乘积先超过数据类型范围，结果就已经错误。

本题中，若两个操作数均小于 (10^9+7)，它们的乘积小于约 (10^{18})，使用 `long long` 是安全的。

### 最终结论

你的核心问题可以概括为：

* **数学计数和手动约分正确；**
* **模数从 (10^9+7) 抄成了 (10^9+9)；**
* **错误地把模后的分子分母做普通除法；**
* **连续乘法没有及时取模，存在整数溢出。**
